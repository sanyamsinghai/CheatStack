What is OOP?

FUNDAMENTALS
Object-Oriented Programming
OOP is a programming style where you organize code around objects — things that bundle together data (attributes) and actions (methods).
Procedural: Code is a sequence of steps.
OOP: Data and functions packaged together inside an object.
The 4 Pillars of OOP
Encapsulation
Keep data safe inside an object. Control how it's accessed.
Inheritance
A child class gets features from a parent class.
Polymorphism
Same method name, different behavior in different classes.
Abstraction
Hide complexity. Show only what's needed.

Classes & Objects

CORE CONCEPT
Class = Blueprint
A class defines the structure. No real data yet — just a template.
class Dog:
    pass
Object = Instance
Creating an object from a class is called instantiation.
dog1 = Dog()
dog2 = Dog()
03

__init__ & self

CONSTRUCTOR
__init__ — The Constructor
Runs automatically when you create a new object. Sets up initial data.
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age  = age
__init__ is called automatically — you never call it manually.
self — The Object Itself
self refers to the current instance. It lets each object keep its own data separate from others.
self.count += 1
self must always be the first parameter of every method.
04

Instance vs Class Variables

VARIABLES
Instance Variables
Belong to a specific object. Each object has its own copy.
self.name = name
Class Variables
Belong to the class itself. Shared by ALL objects. Defined outside __init__.
class Employee:
    company = "TechCorp"
Change a class variable via ClassName.var — not self.var — to affect ALL instances.
05

Methods

INSTANCE · CLASS · STATIC
Instance Method
Most common. Takes self. Can read and change instance data.
def deposit(self, amount):
    self.balance += amount
@classmethod
Takes cls instead of self. Often used as alternative constructors.
@classmethod
def from_string(cls, s):
    return cls(...)
@staticmethod
No self or cls. Just a utility function grouped inside the class.
@staticmethod
def add(a, b):
    return a + b

Encapsulation

DATA PROTECTION
The Idea: Protect Your Data
Encapsulation bundles data and methods and restricts direct access. Python uses naming conventions.
NotationLevelMeaning
namePublicAnyone can access
_nameProtected"Handle with care"
__namePrivateName-mangled — hidden
Python doesn't enforce access like Java/C++. __name becomes _ClassName__name internally.
07

@property — Pythonic Getters/Setters

ENCAPSULATION
Pythonic Way
Access private data like a normal attribute while still having validation.
@property
def age(self):
    return self.__age

@age.setter
def age(self, value):
    if value > 0:
        self.__age = value
@property makes private attributes feel like public ones. Clean and Pythonic.
08

Inheritance

IS-A RELATIONSHIP
Basic Inheritance
A child class automatically gets all attributes and methods from the parent class.
class Animal:
    def speak(self): print("Some sound")

class Dog(Animal):
    pass  # inherits speak() automatically
Method Overriding
A child class can replace a parent method by defining one with the same name.
class Dog(Animal):
    def speak(self):
        print("Woof!")
09

super()

INHERITANCE
Calling the Parent Class
super() lets the child class call a method from the parent class.
class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)
        self.breed = breed
Always call super().__init__() in __init__ to avoid losing inherited attributes.
10

Multiple Inheritance & MRO

ADVANCED
Multiple Inheritance
A class can inherit from more than one parent.
class Flyable:
    def fly(self): print("I can fly!")

class Duck(Flyable, Swimmable):
    pass
MRO — Method Resolution Order
When a method exists in multiple parents, Python uses C3 linearization to decide which one to call.
print(Duck.__mro__)
Python checks MRO left-to-right, depth-first. The first listed class wins.
11

Polymorphism

MANY FORMS
Method Polymorphism
Different classes define the same method name — Python calls the right one for each object.
for a in [Dog(), Cat(), Cow()]:
    a.speak()   # Python picks the right speak() automatically!
Polymorphism lets you write one loop for many different types. No if/elif checking needed.
12

Abstraction

HIDE COMPLEXITY
Abstract Classes with ABC
An abstract class cannot be instantiated. It forces every child class to implement specific methods.
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self): pass
If a subclass doesn't implement all @abstractmethod methods, TypeError is thrown.
13

Dunder / Magic Methods

__SPECIAL__
What are Dunder Methods?
Methods with double underscores. Python calls them automatically in certain situations.
def __str__(self): return f"{self.title}"
def __add__(self, o): return self.value + o.value
def __len__(self): return len(self.items)
Common Dunder Methods
MethodTriggered ByPurpose
__init__ClassName()Constructor — set up object
__str__print(obj)Human-friendly string
__add__obj1 + obj2Addition operator
__eq__obj1 == obj2Equality check
__len__len(obj)Length of object
__getitem__obj[key]Indexing

Composition vs Inheritance

DESIGN PATTERNS
Composition — "Has-A"
Build complex objects from simpler ones. A Car HAS-A Engine.
class Car:
    def __init__(self):
        self.engine = Engine()  # Has-A
General rule: "Favour composition over inheritance."
When to use which?
Inheritance = IS-A. Composition = HAS-A.
AspectInheritanceComposition
CouplingTightLoose
FlexibilityLessMore

Static & Class Methods (Deep Dive)

DECORATORS
Factory Pattern with @classmethod
Provide multiple ways to create an object.
class Temperature:
    @classmethod
    def from_fahrenheit(cls, f):
        return cls((f - 32) * 5/9)
@classmethod enables alternative constructors. Super useful!

Real-World Example

FULL PROJECT
Mini Banking System — All Concepts Combined
Uses: Abstract Class, Encapsulation, @property, Inheritance, super(), Polymorphism.
from abc import ABC, abstractmethod

class Account(ABC):
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.__balance = balance   # private

    @property
    def balance(self): return self.__balance

    def deposit(self, amt):
        if amt > 0: self.__balance += amt

    @abstractmethod
    def account_type(self): pass

class Savings(Account):
    def account_type(self): return "Savings"

s = Savings("Alice", 1000)
s.deposit(500)
print(s.balance)  # 1500

OOP Best Practices

TIPS
Naming Conventions
Follow Python's standard naming rules for readable code.
Classes: PascalCase
Methods/Variables: snake_case
Constants: ALL_CAPS
Private: __ prefix
Common OOP Mistakes
Things that trip up beginners:
1. Forgetting self: All instance methods need self
2. Forgetting super().__init__(): Inherited attrs won't exist
3. Mutable defaults: Never use [] or {} in __init__
4. Over-using inheritance: Favour composition

QUESTION 01
Create a class Car with attributes brand, model, and year. Add a method display_info() that prints all details.
EASY
class__init__method
Define __init__ and store with self. In display_info(), use an f-string to print all three.
class Car:
    def __init__(self, brand, model, year):
        self.brand, self.model, self.year = brand, model, year
    def display_info(self):
        print(f"{self.year} {self.brand} {self.model}")
QUESTION 02
Create a Rectangle class with width and height. Add methods area() and perimeter().
EASY
classmethodsmath
area = width * height. perimeter = 2 * (width + height).
class Rectangle:
    def __init__(self, w, h):
        self.w, self.h = w, h
    def area(self): return self.w * self.h
    def perimeter(self): return 2 * (self.w + self.h)
QUESTION 03
Create a BankAccount class with private __balance. Use @property to safely get/set balance.
MEDIUM
@propertyencapsulationprivate
@property returns self.__balance. @balance.setter validates before updating.
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.__balance = balance
    @property
    def balance(self): return self.__balance
    @balance.setter
    def balance(self, v):
