01

What is OOP?

FUNDAMENTALS
Object-Oriented Programming
OOP is a programming style where you organize code around objects — things that bundle together data (attributes) and actions (methods). Instead of writing a long list of instructions, you model real-world entities.
The 4 Pillars of OOP
Every OOP language is built on these four core ideas:
Encapsulation
Keep data safe inside an object. Control how it's accessed.
Inheritance
A child class gets features from a parent class.
Polymorphism
Same method name, different behavior in different classes.
Abstraction
Hide complexity. Show only what's needed.
02

Classes & Objects

Class = Blueprint
A class defines the structure. No real data yet — just a template.
class Dog:
    pass
Object = Instance
Creating an object from a class is called instantiation.
dog1 = Dog()
dog2 = Dog()
03

__init__ & self

__init__ Constructor
Runs automatically when you create a new object. Sets up initial data.
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age  = age
self — The Object Itself
self refers to the current instance. It lets each object keep its own data separate from others.
self.count += 1
04

Instance vs Class Variables

Instance Variables
Belong to a specific object. Each object has its own copy.
self.name = name
Class Variables
Belong to the class itself. Shared by ALL objects.
class Employee:
    company = "TechCorp"
05

Methods

Instance Method
The most common type. Takes self. Can read and change instance data.
def deposit(self, amount):
    self.balance += amount
@classmethod
Takes cls instead of self. Often used as alternative constructors.
@classmethod
def from_string(cls, s):
    return cls(...)
@staticmethod
No self or cls. Just a regular utility function.
@staticmethod
def add(a, b):
    return a + b

07

@property

Pythonic Getters/Setters
Make private attributes feel like public ones. Clean and Pythonic.
@property
def age(self):
    return self.__age

@age.setter
def age(self, value):
    if value > 0:
        self.__age = value
08

Inheritance

Basic Inheritance
A child class automatically gets all attributes and methods from the parent.
class Dog(Animal):
    def fetch(self):
        print("Fetching!")
Method Overriding
A child class can replace a parent method.
class Dog(Animal):
    def speak(self):
        print("Woof!")
09

super()

Calling the Parent Class
super() lets the child class call a method from the parent class.
class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)
        self.breed = breed
10

Multiple Inheritance

Multiple Parents
A class can inherit from more than one parent.
class Duck(Flyable, Swimmable):
    pass
MRO
Method Resolution Order determines which method is called.
print(Duck.__mro__)
11

Polymorphism

Method Polymorphism
Different classes define the same method name. Python calls the right one for each object.
for a in [Dog(), Cat()]:
    a.speak()
12

Abstraction

Abstract Classes
Forces every child class to implement specific methods.
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self): pass

What are Dunders?
Methods with double underscores. Python calls them automatically in certain situations.
def __str__(self):
    return f"{self.title}"

Composition vs Inheritance

Composition — Has-A
Build complex objects from simpler ones.
class Car:
    def __init__(self):
        self.engine = Engine()
General rule: Favour composition over inheritance.
