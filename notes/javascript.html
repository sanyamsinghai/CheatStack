Variables, Data Types & Memory

FUNDAMENTALS
Variable Declaration
Modern ways to declare variables with different scopes.
var x = 10;    // function-scoped, hoisted
let y = 20;    // block-scoped, hoisted (TDZ)
const z = 30;  // block-scoped, cannot reassign
Primitive Data Types
Number, String, Boolean, Null, Undefined, Symbol, BigInt
typeof 42;           // "number"
typeof "hi";       // "string"
typeof true;       // "boolean"
typeof undefined;  // "undefined"
typeof null;       // "object" (quirk!)
typeof Symbol("id");  // "symbol"
typeof 999n;        // "bigint"
Type Coercion & Equality
Loose vs strict equality, implicit type conversion.
5 == "5";         // true (loose, coerces to same type)
5 === "5";        // false (strict, no coercion)
null == undefined;  // true (loose only)
null === undefined; // false
"" == false;       // true
"" === false;      // false
Boolean("");           // false (falsy values)
Best practice: Always use === and !== to avoid unexpected coercion bugs.
Template Literals
String interpolation and multi-line strings.
const name = "Alice";
const greeting = `Hello, ${name}!`;
const multiline = `
  Line 1
  Line 2
`;

Operators & Control Flow

CONTROL
Logical Operators
AND, OR, NOT, Nullish Coalescing, Optional Chaining
a && b;     // AND
a || b;     // OR
!a;         // NOT
a ?? b;     // nullish coalescing (null/undefined)
obj?.prop;  // optional chaining
arr?.[0];   // optional array access
Spread & Destructuring
Spread operator and destructuring assignment.
// Spread
const arr = [1, 2, 3];
[0, ...arr];  // [0, 1, 2, 3]

// Destructuring
const [a, b, c] = arr;
const {x, y} = {x: 1, y: 2};
const {name: n} = obj;  // rename
if/else vs ternary vs switch
Control flow statements.
// if/else
if (x > 5) { /* ... */ } else { /* ... */ }

// ternary
let result = x > 5 ? "big" : "small";

// switch
switch (type) {
  case "admin": /* ... */ break;
  case "user": /* ... */ break;
  default: /* ... */
}
Loops
for, while, for...of, for...in, forEach.
for (let i = 0; i < 5; i++) { /* ... */ }
while (condition) { /* ... */ }
for (const item of arr) { /* ... */ }
for (const key in obj) { /* ... */ }
arr.forEach((item, idx) => { /* ... */ });

Functions & Scope

FUNCTIONS
Function Declaration vs Expression
Different ways to create functions and their hoisting behavior.
// Declaration (hoisted)
function add(a, b) {
  return a + b;
}

// Expression (not hoisted)
const add = function(a, b) { return a + b; };

// Arrow function (modern)
const add = (a, b) => a + b;
const square = x => x * x;
const fn = () => { /* block body */ };
Arrow functions don't have their own 'this' — they inherit from parent scope.
Default Parameters & Rest Parameters
Set defaults and collect remaining arguments.
function greet(name = "Guest") {
  return `Hello, ${name}`;
}

function sum(...nums) {
  return nums.reduce((a, b) => a + b, 0);
}
Scope: Global, Function, Block
Understanding variable scope and the scope chain.
let global = "outer";  // global scope

function test() {
  let fn_scope = "function";
  
  if (true) {
    let block_scope = "block";  // only here
  }
  
  // block_scope is NOT accessible here
}
Use 'const' by default, 'let' when reassignment needed, avoid 'var'.
Closures
Functions remember variables from outer scope even after execution.
function counter() {
  let count = 0;
  return () => ++count;
}
const inc = counter();
inc();  // 1
inc();  // 2
04

Objects & Prototypes

OBJECTS
Object Creation
Different ways to create and initialize objects.
// Literal
const obj = {name: "John", age: 30};

// Constructor
const obj = new Object();

// Object.create()
const obj = Object.create(null);
Object Methods & Property Access
Working with object properties and methods.
const obj = {name: "Alice", age: 25};

// Access
obj.name;           // "Alice" (dot notation)
obj["name"];       // "Alice" (bracket notation)

// Modify
obj.name = "Bob";
obj.city = "NYC";  // add new property
delete obj.age;    // remove property

// Check
"name" in obj;           // true
obj.hasOwnProperty("name"); // true

// Iterate
Object.keys(obj);      // ["name", "age"]
Object.values(obj);    // ["Alice", 25]
Object.entries(obj);   // [["name", "Alice"], ["age", 25]]
Constructor Functions & Classes
Creating objects with constructors and ES6 classes.
// Constructor function
function Person(name) {
  this.name = name;
}
const p1 = new Person("John");

// ES6 Class
class Person {
  constructor(name) {
    this.name = name;
  }
}
Prototypes & Inheritance
Prototype chain and object inheritance.
function Animal(name) { this.name = name; }
Animal.prototype.speak = function() {
  console.log(this.name + " makes a sound");
};

function Dog(name) { Animal.call(this, name); }
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;

// Or with classes:
class Dog extends Animal {
  speak() { console.log(this.name + " barks"); }
}
05

Arrays & Methods

ARRAYS
Array Creation & Access
Creating arrays and accessing elements.
const arr = [1, 2, 3];
const arr = new Array(5);  // length 5
arr[0];              // first
arr[arr.length - 1];  // last
Mutating Methods (change original)
Methods that modify the array in place: push, pop, shift, unshift, splice, reverse, sort
arr.push(4);           // add to end → [1,2,3,4]
arr.pop();            // remove from end → [1,2,3]
arr.shift();          // remove from start → [2,3]
arr.unshift(0);       // add to start → [0,2,3]
arr.splice(1, 1, 99);  // remove 1 at index 1, insert 99 → [0,99,3]
arr.reverse();        // reverse in place
arr.sort();          // sorts as strings by default!
arr.sort((a, b) => a - b); // numeric sort
Non-Mutating Methods (return new array)
Methods that don't modify original: slice, concat, map, filter, reduce, find, includes
arr.slice(1, 3);           // [2,3] (elements at 1,2)
arr.concat([5, 6]);        // [1,2,3,5,6]
arr.map(x => x * 2);     // [2,4,6]
arr.filter(x => x > 1);  // [2,3]
arr.reduce((a, b) => a + b); // 6 (sum)
arr.find(x => x > 2);    // 3 (first match)
arr.includes(2);         // true
arr.join("-");           // "1-2-3"
Array Destructuring
Unpack array elements into variables.
const [a, b, c] = [1, 2, 3];
const [x, , z] = [1, 2, 3];  // skip middle
const [first, ...rest] = [1, 2, 3];  // rest: [2,3]

Async & Promises

ASYNC
Promises
Handle asynchronous operations with .then().catch().finally()
const promise = new Promise((resolve, reject) => {
  if (success) resolve(value);
  else reject(error);
});

promise
  .then(result => console.log(result))
  .catch(error => console.log(error))
  .finally(() => console.log("done"));
Always use .catch() or try/catch to handle rejections.
async/await
Modern syntax for handling promises — cleaner than .then().catch()
async function fetchData() {
  try {
    const response = await fetch("/api/users");
    const data = await response.json();
    return data;
  } catch (err) {
    console.log("Error:", err);
  }
}
Use await only inside async functions. It pauses execution until promise resolves.
Promise Utilities
Promise.all(), Promise.race(), Promise.allSettled()
Promise.all([p1, p2])
  .then(results => { /* both done */ });

Promise.race([p1, p2])
  .then(firstDone => { /* first completes */ });

Promise.allSettled([p1, p2])
  .then(results => { /* all finished, no reject */ });

ES6+ Features

MODERN JS
let & const
Block scoping and immutability.
let x = 5;
x = 10;  // OK, can reassign

const y = 5;
y = 10;  // ERROR, cannot reassign
y.prop = 20;  // OK, can mutate
Template Literals & Tagged Templates
String interpolation and custom string processing.
const name = "World";
`Hello, ${name}!`  // "Hello, World!"

// Tagged template
const html = styled`color: red`;
function styled(strings, ...values) { /* ... */ }
Object Shorthand & Computed Properties
Concise object notation.
const name = "Alice", age = 30;
// Shorthand
const person = {name, age};

// Computed properties
const key = "name";
const obj = {[key]: "value"};  // {name: "value"}
Destructuring: Objects & Default Values
Unpack objects with renaming and defaults.
const {name, age} = {name: "Alice", age: 30};

// Renaming
const {name: n, age: a} = person;

// Defaults
const {name = "Guest", country = "USA"} = person;

// Nested
const {address: {city}} = person;
for...of & for...in Loops
Different iteration approaches.
const arr = ["a", "b", "c"];
// for...of (values)
for (const item of arr) { console.log(item); }

// for...in (keys/indices)
for (const idx in arr) { console.log(idx); }

DOM & Events

BROWSER
DOM Selection & Manipulation
Selecting and modifying DOM elements.
// Selection
document.getElementById("id");
document.querySelector(".class");      // first match
document.querySelectorAll("div");   // all matches

// Manipulation
el.textContent = "text";
el.innerHTML = "<b>html</b>";
el.setAttribute("data-id", "123");
el.classList.add("active");
el.classList.remove("hidden");
el.classList.toggle("dark");
Event Handling
Listening to and handling user interactions.
// addEventListener (preferred)
button.addEventListener("click", (e) => {
  console.log(e.target, e.type);
});

// Event delegation (efficient for many elements)
ul.addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log("Clicked:", e.target.textContent);
  }
});

// Prevent default & stop propagation
e.preventDefault();
e.stopPropagation();
Common Events
Frequently used browser events.
click        // mouse click
change       // input value changed
submit       // form submitted
keydown      // key pressed
mouseover    // mouse over element
scroll       // page scrolled
load         // page/image loaded
DOMContentLoaded  // DOM ready
setTimeout & setInterval
Delayed and repeated execution.
setTimeout(() => {
  console.log("After 1 second");
}, 1000);

const intervalId = setInterval(() => {
  console.log("Every 1 second");
}, 1000);

clearInterval(intervalId);

📝 Practice Questions

1. What's the difference between var, let, and const?
// var: function-scoped, hoisted, can be redeclared
// let: block-scoped, hoisted (TDZ), cannot be redeclared
// const: block-scoped, not hoistable, cannot be reassigned or redeclared

// Best practice: Use const by default, let if you need reassignment, avoid var
2. Explain closures and provide an example
// A closure is a function that has access to variables from its outer scope
// even after the outer function has finished executing.

function outer(name) {
  return function inner() {
    console.log(name);  // 'name' is captured
  };
}
const greet = outer("John");
greet();  // "John" - closure remembers 'name'
3. What is the difference between synchronous and asynchronous code?
// Synchronous: Code executes line by line, waits for each to complete
console.log(1);
console.log(2);  // waits for 1 first
console.log(3);

// Asynchronous: Code doesn't wait, uses callbacks/promises
console.log(1);
setTimeout(() => console.log(2), 1000);  // happens later
console.log(3);  // runs immediately after 1
4. What's the difference between map and forEach?
// map: returns new array, useful for transformations
const doubled = [1,2,3].map(x => x * 2);  // [2,4,6]

// forEach: doesn't return anything, useful for side effects
[1,2,3].forEach(x => console.log(x * 2));  // prints 2, 4, 6

// Use map when you need the result, forEach for side effects
5. Explain event delegation
// Instead of adding listeners to each item, add one to parent
// and check the target

const list = document.getElementById("myList");
list.addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log("Li clicked:", e.target.textContent);
  }
});

// Benefits: Better performance with many elements, dynamic elements work too
